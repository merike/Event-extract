Event-extract by merike

Event-extract was a prototype addon for solving bug 403222. Current Thunderbird includes this functionality by default and event-extract addon is unnecessary.

Extension

You can download a recent version of the standalone extension here.
Testing one for localizers that enables UI to test patterns is available at tricky-bits.eu/event-extract.xpi.
Old testing one is still available at downloads.

Testing on Earlybird

After translating all of calendar.dtd and calendar.properties and at least some of calendar-extract.properties you need to wait for the next nightly build at http://ftp.mozilla.org/pub/mozilla.org/calendar/lightning/nightly/latest-comm-aurora-l10n/ to appear and install it as well as the extension for localizers mentioned above (until UI is not yet available in nightly builds, see bug 886124).

You don't need to have invitations to test. You can write a draft invitation yourself. To add this as an event or task right-click on message header toolbar and add extraction buttons.

The extension will log some data to error console while converting which you might find useful. You can see it by pressing Ctrl+Shift+J

Expected behaviour

For events extract both start and end date and time from email.

For tasks extract both date and time of due date from email and use email Date header as start time.

Language chosen depends on dictionaries installed and locales included in Lightning (nightlies only have one). A dictionary that matched the highest amount of email content determines the patterns used. If no dictionary gives a match of > 50% or no dictionaries are installed then application locale determines the language used.
